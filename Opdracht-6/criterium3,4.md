# 1. Weekelijkse teammeetings
**Afspraak:** Wij als developers team hebben elke week een meeting met de coach en de lead developer om de voortgang van het project te bespreken.

**Details:**
- Tijdsduur: Maximaal 30 minuten
- Frequentie: Wekelijks op dinsdag om 12:00 uur
- Verantwoordelijken: Developer team zorgt voor de agenda, de lead developer geeft feedback en begeleiding waar nodig
- Inhoud: Elke teamlid deelt kort: (1) wat hij/zij de afgelopen week heeft gedaan, (2) wat hij/zij de komende week gaat doen, en (3) eventuele problemen waar hij/zij tegenaan is gelopen.

**Bewijs van de naleving:**
- Notulen van de meetings worden gedeeld in de teamchat.
- Feedback van de coach en lead developer wordt gedocumenteerd en besproken in de volgende meeting.

# 2. Code Reviews
**Afspraak:** Alle code wordt gecontroleerd via pull requests voordat deze in de main branch wordt samengevoegd.

**Details:**
- **Specifiek:** Elke pull request moet minimaal 1 code review hebben van een teamlid.
- **Meetbaar:** Pull requests kunnen pas worden gemerged na goedkeuring van de reviewer.
- **Bereikbaar:** Reviewers hebben maximaal 4 uur om feedback te geven op een pull request.
- **Gedetailleerd:** Zorgt voor een duidelijke code met 1 schrijfstijl en houd iedereen verantwoordelijk voor de codekwaliteit.

**Betrokkenen:** Alle developers.

**Bewijs van de naleving:**
- Pull requests in de repository.

# 3. design ideeën 

**Afspraak:** Voor een nieuw ontwerp of grote wijziging in de UI/UX moet het besproken worden in het team.

**Details:**
- **Specifiek:** Nieuwe ontwerpen of grote wijzigingen moeten duidelijk worden toegelicht met de reden en de verwachte impact.
- **Meetbaar:** De meerderheid van het team moet het eens zijn met het ontwerp voordat het wordt geïmplementeerd.
- **Bereikbaar:** Ontwerpen worden besproken in de wekelijkse meetings of in een apart overleg.
- **Gedetailleerd:** Zorgt ervoor dat het ontwerp aansluit bij de gebruikersbehoeften en de projectdoelen.

**Betrokkenen:** Alle developers.

**Bewijs van de naleving:**
- Notulen van de meetings waarin verbeteringen zijn besproken.
